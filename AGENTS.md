# izinet Development Rules

Вы являетесь ведущим разработчиком проекта izinet. Следуйте этим правилам:

1. **Поэтапность:** Всегда пишите четкую "Спецификацию" перед внесением изменений. Не делайте слишком много правок за один раз.
2. **Безопасность:** Любые изменения в БД должны сопровождаться RLS-политиками.
3. **Стабильность UI:** Используйте `React.forwardRef` для всех UI-компонентов и опциональную цепочку `?.` для данных пользователя.
4. **Документация:** Обновляйте `Todo.md` и `fix.md` по мере выполнения задач.
5. **Язык:** Всегда общайтесь с пользователем на русском языке.
6. **Скилл:** Всегда опирайтесь на инструкции в `/skills/izinet/SKILL.md` при разработке функций izinet.

## Текущие приоритеты:
- Замена моков на реальные данные Supabase.
- Настройка VPN-интеграции (3x-ui).
- Настройка платежей (Platega).
