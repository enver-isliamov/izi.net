### 0. **00_RUN_ME_FIRST.sh** 🎯 (ГЛАВНЫЙ МАСТЕР)

bash /opt/izinet/00_RUN_ME_FIRST.sh

- Интерактивное меню с 8 опциями

**Меню:**
```

[4] 🐳 ПЕРЕСОБРАТЬ DOCKER (10-15 мин)

[6] ✅ ПОЛНАЯ ПРОВЕРКА СИСТЕМЫ
[7] 🔄 ЗАПУСТИТЬ ВСЁ ПО ПОРЯДКУ (авто)
[0] ❌ ВЫХОД
```

**Рекомендация:** Начни отсюда. Просто запусти и выбери [7].

### 1. **01_DIAGNOSTIC_SCRIPT.sh** 🔍

```bash
bash /opt/izinet/01_DIAGNOSTIC_SCRIPT.sh > diagnostic_report.txt 2>&1
```

**Что проверяет:**
1. ✅ Статус Docker контейнеров
2. ✅ Логи izinet-app (последние 100 строк)
3. ✅ Ошибки (grep Error/TypeError/ReferenceError)
4. ✅ Содержимое .env (переменные)
5. ✅ Образы Docker
6. ✅ Прослушивающие порты (3005, 3443, 2053, 443)
7. ✅ Docker сеть и IP контейнеров
8. ✅ Тест запроса к API

**Выход:** Текстовый отчёт с диагностикой

**Когда запускать:**
- Когда что-то не работает
- Перед началом (бейслайн)
- После каждого исправления

### 2. **02_FIX_ENV.sh** 📝 (ИНТЕРАКТИВНЫЙ)

```bash
bash /opt/izinet/02_FIX_ENV.sh
```

**Что делает:**
1. Резервирует старый .env
2. Запрашивает интерактивно:
   - Supabase URL
   - Supabase ANON KEY
   - Supabase SERVICE_ROLE KEY
   - Domain (izinet.online)
   - Telegram Bot TOKEN
   - Telegram Bot Username
   - 3x-UI пароль
   - ENOT Merchant ID
   - ENOT Secret Keys (1 и 2)
3. **Генерирует новые Reality ключи** через `docker exec x3-ui xray x25519`
4. Создаёт новый `/opt/izinet/.env`

**Результат:** `/opt/izinet/.env` с 15+ переменными

**Требует подготовки:**
- ✅ Значения от Supabase (в личном кабинете)
- ✅ Telegram Bot TOKEN (@BotFather)
- ✅ ENOT.io данные (если используешь платежи)

**Когда запускать:**
- Если .env пустой или повреждён
- Если переменные окружения не загружаются (injected env (0))
- Каждый раз при переносе на новый сервер

---

### 3. **03_FIX_CODE.sh** 🔨 (АВТОМАТИЧЕСКИЙ)

```bash
bash /opt/izinet/03_FIX_CODE.sh
```

**Что делает:**
1. ✅ Проверяет и исправляет: `isIPOrEmpty` → `isIpAddr` (ReferenceError fix)
2. ⚠️ Предупреждает о скидочных множителях в ценах (0.95, 0.85, 0.75)
3. 📋 Показывает что нужно исправить вручную

**Автоматические исправления:**
- Fix A: ReferenceError в xui.service.ts
- Fix B: Проверка переменных в коде

**Ручные исправления (если нужны):**
- Замена скидочных множителей на Math.round()
- Исправление логики создания подписок

**Когда запускать:**
- После клонирования репо с известными багами
- Если видишь ReferenceError в логах
- Перед пересборкой Docker

### 4. **04_REBUILD_DOCKER.sh** 🐳 (ДОЛГИЙ)

```bash
bash /opt/izinet/04_REBUILD_DOCKER.sh
```

**Что делает:**
1. 🛑 Останавливает контейнеры: `docker compose down`
2. 🧹 Очищает неиспользуемые образы: `docker image prune -f`
3. 📦 Пересобирает: `docker compose build --no-cache izinet-app`
4. 🚀 Запускает: `docker compose up -d`
5. ⏳ Ждёт 30 сек и показывает логи

**Время:** 10-15 минут (зависит от интернета)

**Результат:** Свежий контейнер с новым .env и исправленным кодом

**Когда запускать:**
- После исправления .env
- После исправления кода
- Когда app крашится

---

### 5. **05_FIX_REALITY_KEYS.sh** 🔐

```bash
bash /opt/izinet/05_FIX_REALITY_KEYS.sh
```

**Что делает:**
1. 🔐 Читает ключи из .env
2. 🔄 Обновляет их в SQLite базе x3-ui
3. 🚀 Перезагружает x3-ui контейнер
4. ✅ Проверяет что ключи совпадают

**Самая важная функция:** Синхронизирует Reality pubkey в `.env` и в x3-ui DB

> **Это часто причина Hiddify таймаутов!**

**Когда запускать:**
- Всегда после 02_FIX_ENV.sh
- Когда Hiddify показывает 404/timeout
- Когда меняешь Reality ключи

## ⚠️ ВАЖНЫЕ ЗАМЕЧАНИЯ

1. **Все скрипты требуют bash** (не sh, не zsh, именно bash)
   ```bash
   bash script.sh    # ✅ Правильно
   ./script.sh       # ⚠️ Проверь shebang
   sh script.sh      # ❌ Может не работать
   ```

2. **Все скрипты нужно запускать из `/opt/izinet`**
   ```bash
   cd /opt/izinet
   bash 00_RUN_ME_FIRST.sh
   ```

3. **Требуется Docker и docker-compose**
   ```bash
   docker --version
   docker compose --version
   ```

4. **Требуется sqlite3** (для чтения x3-ui базы)
   ```bash
   apt install sqlite3
   ```

5. **Скрипты требуют прав root/sudo**
   ```bash
   sudo bash 00_RUN_ME_FIRST.sh  # Или логируйся как root
   ```
