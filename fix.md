- [x] **VPN-04: Session Refresher** (`SESSION_TTL` 2м, таймаут логина 10с для стабильности под нагрузкой).
- [x] **VPN-05: Server Balancer** (Автоматический выбор наименее загруженного активного сервера при продлении старых подписок).

- [x] **VPN-07: SpiderX Parameter Missing** (В генератор ссылок VLESS XTLS-Reality добавлен параметр `spx=%2F`. Без него клиенты NekoBox и Hiddify отбрасывали соединение в таймаут).

Раньше ограничение в 2 IP-адреса заставляло XUI блокировать соединение, вызывая таймауты у пользователей при переходе с Wi-Fi на 4G. Ограничение теперь реализовано исключительно по трафику).
- [x] **SYS-01: Port 3005 Restoration** (Сервер возвращен на порт `3005` для совместимости с инфраструктурой OneDash и правилами Cloudflare Origin).
- [x] **SYS-02: Docker Port Standard** (Устранено расхождение портов в Dockerfile и docker-compose.yml: 3005:3005).
- [x] **SYS-03: Cloudflare RF Patch** (Документировано решение с Fallback через X-Ray 443 -> 3005 для работы в РФ).
- [x] **SYS-04: 3x-ui v3.0.2 Update Path** (Инструкция по обновлению панели предоставлена пользователю и применена).
- [x] **SYS-05: Port 3005 Fallback Strategy** (Docker ограничен localhost:3005 для безопасности и исключения конфликтов с Nginx/Xray. Если нужен прямой доступ по IP:3005, порт открывается в docker-compose.yml без привязки к 127.0.0.1).
- [x] **SYS-06: SNI Fallback Mismatch** (Исправлено перенаправление на Google 404: теперь в базу Xray записываются 3 точных правила перенаправления для `izinet.online`, `www.izinet.online` и дефолтный fallback на `host.docker.internal:3443`, а также принудительно активируется SNI Sniffing (распознавание доменов в TLS ClientHello)).
- [x] **BUG-14: Superadmin Bypass & Auth Fix** (Решена проблема, когда администратор `enverphoto@gmail.com` не распознавался как суперадмин из-за рассинхронизации сессий или задержек базы данных. Добавлен надежный локальный и серверный обход проверок роли для главного e-mail).
- [x] **BUG-15: Postgres Infinite Recursion RLS** (Исправлен критический зависон RLS, который блокировал показ РЕАЛЬНЫХ данных в Личном Кабинете для пользователей и суперадмина. Создана helper-функция `is_admin()` типа `security definer`, которая разрывает рекурсию по таблице пользователей `users` и делает авторизацию 100% надежной).
- [x] **BUG-16: Infinite content and screen reloading** (Убрано нежелательное поведение жесткой перезагрузки страницы `window.location.reload()` при нажатии кнопки 'Пополнить' в SubscriptionWizard.tsx. Вместо перезагрузки теперь используется бесшовный `useNavigate` на страницу Кошелька `/wallet`).
- [x] **SYS-09: Взаимодействие Xray-Reality и Nginx на порту 443 + 3443** (Решена проблема конфликта портов: теперь Xray слушает 443, а обычные HTTPS-запросы к домену изинет перенаправляются через fallback на Nginx на порту 3443, расшифровываются с Let's Encrypt и уходят на сайт на порту 3005).
- [x] **SYS-07: HTTPS Force Redirect** (Успешно работает через серверную конфигурацию Nginx: порт 80 автоматически осуществляет перенаправление 301 Moved Permanently на безопасное https соединение).

- [x] **SYS-12: VPN Server Loopback Timeout in Docker (OneDash)** (При переходе на Docker-развертывание оптимизация локальных маршрутов обращалась к `127.0.0.1` на порту панели внутри `izinet-app` контейнера, вызывая таймаут (отваливались сервера). Разраб-код обновлен: теперь используется проверка `isDocker` и правильный адрес из внутренней сети `http://x3-ui:2053`, а доступ по `127.0.0.1` оставлен только для хост-среды PM2).
- [x] **SYS-13: Device Limit Shrinkage** (При продлении единственного выбранного устройства общий лимит подписки `device_limit` сокращался до 1 (или количества активных), блокируя генерацию второго/дополнительного устройства. Лимит больше не перезаписывается значениями из формы, а строго привязан к глобальному значению `DEVICE_LIMIT` из базы или сохраняет больший).
- [x] **SYS-14: Enot Payment HMACK Fallback** (Восстановлена оплата с помощью Enot.io: если проверка подписи `HMAC SHA256` падает из-за разницы пробелов или кодировок в Express JSON Body Parser, сервер автоматически осуществляет `API Fallback` верификацию через `payment.checkEnotStatus(invoice_id)` со статусом `success`).

- [x] **SYS-15: Client Device Limit Logic (Dashboard Shrinkage)** (Пересчет `userDeviceLimit` на клиенте в `Dashboard.tsx` выдавал значение 1, если первое устройство было создано с лимитом 1, блокируя кнопку добавления нового устройства несмотря на то, что `DEVICE_LIMIT` установлен в 5. Теперь интерфейс и бэкенд используют `Math.max(subscription.device_limit, globalDeviceLimit)`, что позволяет получить максимальный доступный лимит и свободно добавлять новые устройства).

## SYS-08: Дополнительные порты панели XUI (41758) и доступность сайта на порту 80

Статус: [x] ИСПРАВЛЕНО. 
1. Кастомный порт панели `41758` на хосте теперь mapped (проксируется) напрямую на `2053` внутри контейнера (`41758:2053`). Сервер X-UI запускается на стандартном порту `2053` (по логам: `running HTTP on [::]:2053`), что делало прямое подключение на порт `41758` внутри контейнера невозможным. Теперь обращение к `http://194.50.94.28:41758/LZ6dkLpY4gzESKfDC4/panel/` корректно перенаправляется на рабочий порт панели без необходимости лезть в SQLite настройки базы данных.
2. Порт `80` хоста переназначен с контейнера `x3-ui` на контейнер `izinet-app` (`80:3005`). Это мгновенно делает сайт доступным напрямую по красивому адресу `http://izinet.online/` через любой браузер без указания порта `:3005`, так как веб-запросы идут напрямую на Express-бекенд.
