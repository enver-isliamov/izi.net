## Что вам нужно реализовать сейчас (Технический план)

Если цель — **настоящая бесшовность и балансировка (Smart Balance)**, то план действий следующий:

### 1. Синхронизатор (Supabase -> X-UI API)
В текущем состоянии `izinet` не дублирует клиента на запасной сервер, а держит его на том, который был назначен (через `server_id`).
- **Как изменим:** Нам потребуется переписать логику добавления клиента (`/api/subscription/buy`, и скрипты `check_realtime/check_servers`), чтобы мы рассылали POST-запросы `addClient` (с одинаковым email и UUID) на **все** активные серверы.
- Это обсуждаемо — как только скажете "Делаем единую базу на всех серверах", я перепишу backend за несколько шагов.

### 2. Бесшовность через Reality (Настройка панелей X-UI)
Чтобы переключение IP в телефоне было незаметным (без разрыва TCP/Reality сессии), **вручную в X-UI** на обоих серверах сделайте:
1. **Master Key:** Сгенерируйте 1 пару ключей (Private/Public) и поставьте на все сервера.
2. **Dest & Server Names:** Укажите один домен (например, `dl.google.com`).
3. **Short IDs:** Обязательно одинаковые.
*(Код izinet это не делает, это инфраструктурная часть).*

### 3. Формирование ссылки для пользователя
Формат ссылок уже поддерживается `izinet`. При переходе на дублирование серверов, наша ручка `/api/sub/:id` для одного устройства начнет отдавать список (Base64) такого вида:
```text
vless://uuid@185.72.11.57:443?security=reality&sni=google.com&fp=chrome&pbk=ВАШ_КЛЮЧ&sid=ВАШ_ID#Server_1_Main
vless://uuid@194.50.94.28:443?security=reality&sni=google.com&fp=chrome&pbk=ВАШ_КЛЮЧ&sid=ВАШ_ID#Server_2_Reserve
```
Клиенту останется только включить Auto-Test или Load Balance в своем приложении.

### 4. Обход "Белых списков" (Мобильный интернет РФ)
Это чисто инфраструктурная задача:
- Только `443` порт.
- Хорошие SNI (`update.microsoft.com`).
- Flow: `xtls-rprx-vision` (обход детектирования по отпечатку TLS).

> **ВЫВОД:** Документ RECO.MD — это отличный план. С точки зрения архитектуры проекта `izinet`, база уже готова. Остался один шаг: если вы хотите бесшовный автоматический переход, нам нужно будет отменить жесткую привязку `device <-> 1 server` и переписать код так, чтобы мы "размазывали" подписки пользователей по **всем доступным серверам сразу**. Дайте команду, если согласны перестроить логику.

### 5. Оптимизация Cloudflare и исправление ошибки "Invalid URL"

Ошибка **"Invalid URL"** (Akamai/Edgesuite) на вашем скриншоте возникает, когда домен проксируется через инфраструктуру, которая не ожидает Reality-трафика.

**Рекомендуемый алгоритм исправления:**

#### Шаг 1: DNS (Cloudflare)
1. Зайдите в панель Cloudflare.
2. В разделе **DNS** найдите запись для `izinet.online`.
3. Установите **Proxy Status** в режим **"DNS Only"** (Серое облако). 
   *Это критично: Reality работает только при ПРЯМОМ соединении с сервером. Если облако оранжевое, Cloudflare пытается «быть сервером» сам и ломает соединение.*

#### Шаг 2: Настройка Reality (Panel 3x-ui)
Зайдите в `Подключения -> Reality (443) -> Изменить` и проверьте следующие поля:
1. **Dest (Цель маскировки):** Укажите **ЧУЖОЙ** домен (например, `dl.google.com:443`). 
   *Никогда не указывайте здесь свой домен `izinet.online`, иначе X-Ray зациклится при попытке перенаправить трафик.*
2. **Server Names (SNI):** Укажите тот же домен (`dl.google.com`).
3. **Раздел Fallbacks (внизу):**
   - Нажмите **+ Add**.
   - **SNI:** `izinet.online` (только здесь пишем ваш реальный домен).
   - **Path:** (оставить пустым).
   - **Dest:** `3005` (или `127.0.0.1:3005`). Это перенаправит «чистый» HTTPS трафик на наш сайт.

#### Шаг 3: Перезапуск
Нажмите **Сохранить** и обязательно **Перезапустить Xray** в панели.

### 6. Работа с подписками (Custom Path)
Вы указали в панели путь `/izisub/`. Это влияет только на встроенные ссылки панели 3x-ui. 
Скрипт `izinet` продолжит использовать `/api/sub/:id` нашего сервера, так как мы сами формируем список конфигов для клиента. Смена пути в панели ничего не сломает для основного приложения.

Изучив документацию вашей панели (v3.0.2), вот что мы внедрим в следующих этапах:

1. **Auto-Health Check:** Добавим в админку izinet статус серверов в реальном времени (Liveliness) через API панели.
2. **One-Click Update:** Кнопка в админке, которая сама пулит конфиги и обновляет ссылки у пользователей.
3. **Smart Subscriptions:** Использование `WebBasePath` из настроек панели для автоматической генерации защищенных ссылок подписок.
4. **Traffic Webhooks:** Реализовано API для приема уведомлений о лимитах. 
   - **Webhook URL:** `https://izinet.online/api/webhooks/traffic-limit?token=izinet_secret`
   - **Где настроить:** В панели 3x-ui в разделе "Telegram Bot" или в скрипте уведомлений. Панель должна отправлять POST запрос с JSON: `{ "email": "clientX", "traffic_used": 123, "traffic_limit": 1000 }`.
   - **Результат:** Мгновенная блокировка просроченных подписок без ожидания фоновой синхронизации.
