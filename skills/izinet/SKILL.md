# Скилл izinet: Стандарт Разработки

Этот скилл предназначен для эффективной разработки проекта izinet. Он минимизирует расход токенов, фокусируя внимание на текущей задаче, и предотвращает технические ошибки.

## 1. Режим работы (Lifecycle)
Всегда работай по следующему циклу:
1.  **Анализ Намерения:** Четко определи, чего хочет пользователь (баг, фича, информация).
2.  **Спецификация (Step-by-Step):** Перед написанием кода выведи план изменений на русском языке.
3.  **Изолированная реализация:** Изменяй только те файлы, которые необходимы.
4.  **Проверка (Lint/Compile):** Обязательно проверяй проект на ошибки после правок.
5.  **Отчет:** Кратко опиши, что сделано и что делать дальше.

## 2. Архитектурные принципы izinet
*   **Стек:** Frontend (React + Vite + Tailwind), Backend (Supabase), UI (Base UI / shadcn).
*   **Безопасность (RLS):** Вся работа с данными через Supabase должна сопровождаться SQL-миграциями с настройкой RLS. Никогда не разрешай "публичный" доступ (`allow all`) без веской причины.
*   **Опциональная цепочка (Optional Chaining):** Всегда используй `?.` при работе с объектом `user` в React-компонентах, чтобы избежать "черного экрана".
*   **Ref Forwarding:** Все UI-компоненты (кнопки, меню) ДОЛЖНЫ поддерживать `React.forwardRef`.

## 3. Работа с данными (Supabase)
При создании новых таблиц:
1.  Создавай файл миграции `supabase/migrations/[timestamp]_name.sql`.
2.  Включай `ALTER TABLE name ENABLE ROW LEVEL SECURITY;`.
3.  Создавай политику: `CREATE POLICY "Users can access their own data" ON public.name FOR ALL USING (auth.uid() = user_id);`.

## 4. Гайд для новичка
*   Объясняй сложные концепции (например, RLS или Hydration) простым языком.
*   При возникновении ошибок (например, 504 или Quota Exceeded), честно говори пользователю, что это ограничение инфраструктуры, а не баг кода.

## 5. Контекст проекта
*   **Файлы документации:** Всегда сверяйся с `Todo.md` и `fix.md`.
*   **Цель:** Преобразование моковых данных в реальные данные из Supabase.
*   **Интеграции:** VPN (3x-ui), Платежи (Enot.io).

## 6. Триггеры скилла
Используй этот гайд если:
*   Пользователь упоминает "izinet".
*   Нужна новая фича или фикс.
*   Нужна миграция БД.
*   Вопрос про архитектуру или деплой.
*   Пользователь спрашивает "что дальше?".
